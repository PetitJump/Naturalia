Complexité algorithmique

Modèles mathématiques et structures de données qui régissent la simulation

Naissances
01
Reproduction probabiliste par couples O(n)
Les individus dont l'âge dépasse maturite_sexuelle sont filtrés puis regroupés en n // 2 couples. Chaque couple génère un nombre aléatoire de petits dans la fourchette [min, max] définie dans data.json. Chaque nouveau-né passe ensuite un test de survie juvénile indépendant. 
pour chaque couple dans adultes // 2 :
    petits = randint(min_nee, max_nee)
    pour chaque petit :
        si random() < 0.50 :    # mortalité juvénile 50 %
            population.append(Individu(age=0))
Complexité : O(n × max_nee) — linéaire en fonction de la population min_nee, max_nee — fourchette de petits par couple, configurable dans data.json
02
Herbe — équation logistique de Verhulst (1838) O(1)
L'herbe ne se reproduit pas par couples. Sa croissance suit le modèle logistique : rapide quand la population est faible, nulle quand elle atteint la capacité maximale K. C'est une équation différentielle discrétisée sur des pas annuels. 
N          = len(vegetaux)
croissance = int( r × N × (1 − N / K) )
croissance = max(0, croissance)    # évite les valeurs négatives
r — taux de croissance intrinsèque (0.15 lent / 0.25 normal / 0.40 rapide) N — population actuelle de touffes d'herbe K — capacité de charge de l'environnement (plafond souple) Quand N → K, le facteur (1 − N/K) → 0 : la croissance s'annule naturellement
03
Natalité des cerfs conditionnée à l'herbe disponible
La reproduction des cerfs est modulée par la pression alimentaire instantanée. Si l'herbe est abondante, chaque couple tente de se reproduire normalement. En dessous du seuil de confort, le taux de natalité chute proportionnellement. 
herbe_par_cerf = len(vegetaux) / max(1, nb_cerfs)
facteur_herbe  = min(1.0, herbe_par_cerf / 5.0)

# facteur = 1.0 → repro pleine  (≥ 5 touffes / cerf)
# facteur = 0.4 → 40 % des couples se reproduisent
# facteur = 0.0 → aucune naissance

si random() > facteur_herbe :
    passer    # ce couple ne se reproduit pas
Morts — 4 mécanismes indépendants
01
Prédation — loups sur cerfs O(1)
Chaque loup chasse combien cerfs par an (paramètre modifiable dans data.json). Si les cerfs disponibles sont insuffisants pour nourrir toute la meute, les loups en excès meurent de faim. 
cerfs_tues    = nb_loups × combien
cerfs_tues    = min(cerfs_tues, nb_cerfs)

loups_nourris = cerfs_tues // combien
loups_affames = nb_loups − loups_nourris
# les loups_affames sont retirés de la liste
02
Broutage progressif — cerfs sur herbe O(1)
Trois cas selon la disponibilité de l'herbe. La mortalité n'est pas binaire : elle est proportionnelle au manque via un taux de satisfaction, ce qui évite les extinctions brutales et rend la simulation plus réaliste. 
touffes_requises = nb_cerfs × combien

si herbe ≥ touffes_requises :
    herbe -= touffes_requises     # broutage normal

sinon si herbe > 0 :
    taux   = herbe / touffes_requises
    survie = 0.30 + 0.70 × taux   # mortalité proportionnelle
    cerfs  = [c pour c si random() < survie]
    herbe  = []

sinon :                           # plus d'herbe du tout
    cerfs  = [c pour c si random() < 0.20]  # 80 % meurent
03
Mortalité naturelle aléatoire O(n)
Chaque individu tire sa survie annuelle indépendamment des autres. Le taux dépend uniquement de l'âge. Implémenté via une liste en compréhension : une seule passe sur la population, sans tri ni structure auxiliaire. 
taux = 0.50 si age == 0 sinon 0.95

population = [ind pour ind dans population
              si random() < taux(ind.age)]

# Complexité : O(n) — une seule passe sur la liste
04
Mort par vieillesse O(n)
L'âge de chaque individu est incrémenté à chaque appel de update(). Les individus dépassant l'âge maximal sont filtrés lors de la même passe que la mortalité naturelle. Chaque individu est une instance de classe avec un attribut .age propre. 
loups = [k pour k dans loups si k.age < 20]
cerfs = [c pour c dans cerfs si c.age < 12]

# k.age est incrémenté à chaque update() dans naissance()
# Loups : âge max 20 ans — Cerfs : âge max 12 ans
Événements météo
01
Tirage séquentiel avec arrêt anticipé O(k)
Les événements sont parcourus séquentiellement depuis data/meteo.json. Pour chaque événement, un flottant aléatoire est comparé à sa probabilité. Le premier événement déclenché stoppe la boucle. Le système est extensible : ajouter un événement dans le JSON suffit, sans modifier le code Python. 
pour cle, ev dans meteo.items() :
    si random() < ev["chance"] :
        appliquer_meteo(ev)
        break    # un seul événement par année

# k = nombre d'événements dans meteo.json (actuellement 5)
# Dans le pire cas : O(k) — dans le cas moyen : O(1) car peu probable
